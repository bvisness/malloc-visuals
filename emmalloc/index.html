emmalloc demo

Slabs

emmalloc allocates contiguous "slabs" of memory via sbrk. These slabs are broken down into Regions, which may be either used or free. These slabs are also prefixed with a RootRegion, which tracks the overall size of the slab and forms a singly-linked list with other slabs of memory.

Because emmalloc assumes that it is not the only user of sbrk, consecutive slab allocations may not be contiguous. If a new slab is contiguous with the last, emmalloc will grow the current slab instead. Otherwise, the new slab will be added to the linked list of slabs instead.
